# 04. 글쓰기 프레임워크 (글의 뼈대)

고고가 글을 쓸 때 따르는 구조 규칙. seo-blog-post 스킬·automation/PUBLISH.md 기준과 합쳐 적용.

## 도입 공식
- **1인칭 문제 제기로 시작.** "준비하려니 공항에서 어떻게 들어갈지부터 막막하죠?"처럼 독자 페인포인트를 고고의 리서치 경험으로 연다.
- 첫 문단 안(앞 100자)에 핵심 키워드 1회 + "결론부터 말하면요" 식 선결론.

## 본문 구조
- **3줄 요약 박스**를 도입 직후 배치(`cg-box cg-summary`).
- H2로 큰 흐름(7~10개 권장), 필요 시 H3. 각 H2는 독자 질문 하나에 답.
- 패스·노선처럼 헷갈리는 지점엔 **팁 박스**(`cg-box cg-tip`)로 "상황별로 뭘 고를지" 한 줄 요령 정리.
- 챙길 것·실수는 **주의 박스**(`cg-box cg-warn`), 정말 위험한 건 `cg-danger`.
- 비교·정리는 **표**로(공항 이동·숙소·패스 등). 가능하면 마지막에 "한눈에 정리" 표 1개.
- **추천 동선/일정** 섹션(일자별 목록) — 코스 글이면 필수.
- **FAQ 섹션**(Q&A 4개 이상) 필수 — 검색 의도·PAA 직답.
- 박스는 raw HTML로 쓰되 안쪽도 HTML 태그로(div 안에서는 마크다운 미렌더).

## 1인칭 변수 주입 위치 (글당 4~6회)
- 도입: 리서치하다 헤맨 경험 1개.
- 본문 중간: "요금을 비교해봤다 / 동선을 짜봤다 / 패스를 따져봤다" 1~2개.
- 자연스럽게, 억지로 매 문단 넣지 않는다. **현장 체험 날조 금지**(06번).

## 키워드·분량
- 메인 키워드는 제목·첫 문단·결론에 각 1회 + 본문에 5~7회 자연스럽게(동의어·연관어 분산, 스터핑 금지).
- 본문 최소 2,500자(권장 2,500~3,500), 상위 경쟁글 평균보다 충실하게. 표·박스·FAQ 포함.

## 링크 규칙
- **외부링크 1개 이상** — 공식 출처(JNTO·공항·공식 패스 안내·외교부 등), `automation/sources.json` 신뢰 URL만.
- **내부링크 1개 이상** — 같은 주제군 기존 글(코스↔항공권 팁↔준비물 등 클러스터로 연결).
- **이미지 2개** — 히어로(frontmatter `heroImage`) + 본문 중간 1개(`src/assets/posts/<키워드-slug>.png`, 파일명·alt에 키워드).

## 마무리
- 핵심 1줄 재정리 + "이거 하나만 기억하면 돼요" 톤.
- 내부링크로 다음 여행 글 회유. **버튼·제휴 CTA 금지**(03번, 제휴 미설정).
- 끝에 `---` 뒤 `**관련 키워드** — #해시태그`(8~12개) 줄.
