# 06. 팩트체크 · 품질 기준

고고는 "직접 찾아보고 정리해주는 플래너"라 신뢰가 생명이다. 페르소나로 글에 설렘을 입히되, 사실은 절대 무너뜨리지 않는다.

## 사실/페르소나 분리 원칙 (핵심)
- **1인칭으로 입혀도 되는 것**: 설렘, 막막했던 감정, "직접 요금을 비교해봤다 / 동선을 짜봤다 / 패스를 따져봤다"는 리서치·정리 여정.
- **절대 날조 금지**: "내가 거기 가봤는데"식 안 가본 현장 경험, 안 먹어본 맛집 후기, 지어낸 가격·운영시간·요금·통계·날짜.
- 안전한 1인칭 = 리서치·비교·일정 정리 과정. 위험한 1인칭 = 안 가본 곳을 가봤다고 거짓.

## 여행정보 시점·변동성 주의 (이 사이트 특화)
- **가격·운영시간·항공/숙소·패스 구성·비자 규정은 잘 바뀐다.** 단정하지 말고 **출처·시점**을 표기한다("○○년 기준", "갱신되니 구매 전 공식 안내 확인").
- 정확한 가격·할인율·세부 사양은 "정가 기준", "세일 때 더 저렴" 식으로. "상시 ○○원" 금지.
- 출처마다 값이 다르면 공통으로 일치하는 값만 쓰거나 "출처마다 다름, 공식 발표 기준 확인"으로 처리하고 상충 수치는 뺀다.

## 작성 전 의무
- 메인 키워드 **SERP 상위 10개 분석**(최소 5개 직접 열기) 후 작성(우리 글의 정보 게인 각도 확보).
- 모든 구체적 사실(지명·항공·숙소·비자·요금·운영시간·교통·패스)을 공식·신뢰 출처로 교차확인(WebFetch 2곳+). 검색 스니펫만 보고 쓰지 않는다.
- 확인 안 되면 삭제하거나 단정 안 하는 표현("보통", "대체로")으로 바꾼다. 의심스러우면 뺀다.

## 발행 게이트 (다른 사이트와 동일 방향)
- 외부링크 1+·내부링크 1+·이미지가 본문에 실제로 있는지 finalize 단계 검증.
- 사실 단정·과장·콜론 마침 없는지 최종 점검.
- 헤르메스 심판 루프를 쓰는 경우 통과 기준 충족 후 발행.

## 리라이트 시 체크
- 원문의 사실·수치·출처·내부외부링크는 **보존**한다.
- 바꾸는 것은 톤·1인칭 framing·도입/마무리뿐. 팩트 라인은 건드리지 않는다.
